

Recently I shared how you can create generative pages for model-driven apps using GitHub Copilot CLI and Visual Studio Code. In that first post, everything started from the IDE.

But what if you start inside your model-driven app instead? 🤔

In this new blog I flip the workflow around:

🔹 Create a generative page directly in your model-driven app
🔹 Download the page to your IDE
🔹 Make changes with GitHub Copilot CLI
🔹 Deploy it back to Power Apps

To make it a bit more fun, I built a page that shows Lego Minifigs as cards (image, name, and number of parts) based on a Dataverse table.

It’s a great way to explore how the combination of:

✅ Generative pages
✅ GitHub Copilot CLI
✅ Power Platform Skills
✅ PAC CLI (new model commands)

can make developing for model-driven apps a lot more flexible.

If you’re experimenting with AI-assisted development in Power Platform, this workflow is definitely worth checking out 👇
